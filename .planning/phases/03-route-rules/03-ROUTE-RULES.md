# Phase 3 Route Rules

> Pattern source: `02-POI-STRATEGY`.
> This document follows the same "5 组规则 + 执行要点 + 决策追踪表" pattern for Phase 3.

## 1. 路线顺序与分段生成（D-10 / D-11 / D-12）

- **D-10（ROUTE-01）**：路线主链路固定为 `起点 -> 途经点 -> 终点`。
- **D-11（ROUTE-01）**：允许回路/往返段，但需要在正文明确标注。
- **D-12（ROUTE-01 / ROUTE-03）**：非连续重复点保持原顺序，并追加“可能绕路”提示。

执行要点：
- 仅合并连续重复点，保留非连续重复点。
- 输出正文必须能让用户看出每段衔接关系和回路位置。

## 2. 分段交通方式输入与分配（D-01 / D-02 / D-03 / D-04 / D-05）

- **D-01（ROUTE-02）**：按路段的交通方式遵循“用户指定优先”。
- **D-02（ROUTE-02）**：支持用户只对部分路段指定方式。
- **D-03（ROUTE-02）**：未指定路段统一补为默认骑行。
- **D-04（ROUTE-02）**：指定与常识冲突时不改写，仅追加风险说明。
- **D-05（ROUTE-02）**：分段交通方式输入采用自然语言优先，先解析自然语言，再落到段级结构。

执行要点：
- 解析自然语言时保留原句，输出“原句 + 解析结果 + 未命中段默认骑行”。
- 段级字段建议最少包括：`segment_index`、`mode`、`mode_source`。

## 3. 起终点缺失兜底（D-06 / D-07 / D-08 / D-09）

- **D-06（ROUTE-03）**：起点缺失时，兜底为城市中心。
- **D-07（ROUTE-03）**：终点缺失时，兜底为最后一个候选 POI。
- **D-08（ROUTE-03）**：起终点都缺失时，同时应用两侧兜底。
- **D-09（ROUTE-03）**：兜底触发后，A 段与正文必须双重声明来源。

执行要点：
- A 段先声明假设来源，正文在相应分段处再次提示。
- machine-readable 附录应保留 `origin_source` / `destination_source` 便于复核。

## 4. 回路与重复点处理（D-10 / D-11 / D-12）

- **D-10（ROUTE-01）**：连续重复点可压缩为单点。
- **D-11（ROUTE-01）**：回路意图可保留，不做静默删除。
- **D-12（ROUTE-03）**：非连续重复导致明显绕路时，保留并提示“可能绕路”。

执行要点：
- 回路判断要输出 `loop_flag`，不要只在文案层隐含。
- 风险提示优先短句，不替用户做路线重排决定。

## 5. 时间预算冲突处理（D-13）

- **D-13（ROUTE-03）**：当回路意图与时间预算冲突时，时间预算优先。

执行要点：
- 允许弱化回路，但必须说明“为何调整”与“调整后影响”。
- 输出中固定 `route_adjustment=budget_precedence`，避免口径漂移。

## A-G 输出结构映射

- **A=** 假设与兜底声明（起终点缺失来源、自然语言解析状态）
- **B=** 路线摘要（起点、途经、终点与总览）
- **C=** 分段明细（每段 mode、duration_note、mode_source）
- **D=** 风险提示（可能绕路、输入冲突、低置信段）
- **E=** 预算/回路调整说明（时间预算优先的调整结果）
- **F=** 边界声明（本阶段职责与下游接口边界）
- **G=** machine-readable 附录（decision_refs、segment_mode_input、loop_flag 等）

## Phase Scope Boundary

- 本阶段不生成二维码。
