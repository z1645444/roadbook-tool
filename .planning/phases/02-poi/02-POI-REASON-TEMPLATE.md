# Phase 2 POI Reason Template

## Scope

本模板用于 Phase 2 的 POI 推荐理由输出，服务于 D-09 / D-10 / D-11 / D-12，并补齐 D-18 / D-19 / D-20 的低置信机读表达。默认每个候选只输出单句理由，不强制解释 why-not-others。

## 单句模板规则

- 固定表达：`匹配点 + 场景价值`（D-10）。
- 输出形态：每个候选 `单句` 理由（D-09）。
- 默认行为：省略 why-not-others 说明（D-11）。
- 低置信规则：仅在低置信时追加低置信标记（D-12）。

## Template IDs

R1. 匹配点：该地点与用户核心意图高度一致；场景价值：可直接满足当前行程目标并减少无效绕行。

R2. 匹配点：该地点与已给出的偏好（品类/氛围/预算）一致；场景价值：在同城范围内提供更稳定的体验预期。

R3. 匹配点：该地点在当前城市语义相关度更高；场景价值：有利于后续路线编排保持顺序连贯。

R4. 匹配点：该地点与时间窗口或停留节奏更匹配；场景价值：可降低行程拥挤度并提升执行可达性。

R5. 匹配点：该地点与既有候选形成互补类别；场景价值：提升 Top3 多样性并减少同类扎堆风险。

## 低置信变体（仅低置信场景启用）

- 输出格式：`[低置信] {对应 Rn 的单句理由}`。
- 低置信触发后，每个候选都应带低置信标记（D-18）。
- 同时附带 1-2 条`补充信息建议`（D-19），并携带`risk_note`（D-20）。

低置信示例：
- `[低置信] 匹配点：该地点与用户核心意图部分一致；场景价值：可作为临时候选维持路线编排连续性。`

补充信息建议（1-2 条）：
1. 请补充更精确的目的地关键词（如商圈/地标/街道）。
2. 请补充时间窗或交通偏好（如步行优先/换乘可接受范围）。

## Requirement Traceability

- D-09：单候选单句理由。
- D-10：理由固定为匹配点 + 场景价值。
- D-11：why-not-others 默认可省略。
- D-12：低置信标记仅在低置信场景显示。
- D-18：低置信场景不阻断输出，逐项标记。
- D-19：低置信追加 1-2 条补充信息建议。
- D-20：低置信允许下游继续，但需 risk_note。
- POI-02：候选理由必须可解释且格式一致。
