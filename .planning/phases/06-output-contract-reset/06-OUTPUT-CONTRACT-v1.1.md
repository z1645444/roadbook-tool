# 06 输出主契约 v1.1

## 1. 主契约定位

本文件定义 v1.1 的唯一主契约（single source of truth）。
主契约只允许四个一等字段，且必须按固定顺序输出：

1. `route_summary`
2. `address`
3. `eta`
4. `notes`

除以上四字段外，任何字段均不属于 v1.1 主契约。

## 2. 字段定义（唯一主契约）

### `route_summary`

- 含义：路线摘要与行程概览。
- 约束：必须存在，建议为简洁可读文本。

### `address`

- 含义：可复制、可打开的高德地址结果。
- 约束：必须存在，优先输出 `https://uri.amap.com/...`。
- URI 兼容要求：必须包含 `callnative=1`，并保持 URL 编码正确。
- 继承说明：地址语义延续 Phase 5 的输出协议能力，但不回引二维码主字段。

### `eta`

- 含义：预计到达/耗时信息。
- 约束：必须存在；若信息缺失，使用降级说明但字段仍需保留。

### `notes`

- 含义：上下文提示、降级原因或边界说明。
- 约束：必须存在；可承载 degraded 场景说明。

## 3. 固定顺序声明

所有实现、模板、示例与校验输出必须严格遵循以下固定顺序：

`route_summary -> address -> eta -> notes`

不允许重排，不允许插入额外主字段，不允许以兼容层字段替代主字段。

## 4. 禁用字段清单（v1.1 removed）

以下字段在 v1.1 主契约中标记为 `removed`，不属于主契约：

- `qr`
- `qr_status`
- `qr_payload_text`
- `qr_image`
- `ascii_qr`

说明：二维码能力仅允许在后续兼容层中讨论，不得进入 v1.1 主契约。

## 5. 边界约束

- 当前阶段不执行真实请求。
- 当前阶段不发送消息。
- 当前阶段不生成真实二维码图片。
- A-G 结构在 v1.1 中不是主契约来源。

## 6. 需求追踪（OC-01 / OC-02）

| Requirement | 条款映射 | 判定方式 |
|-------------|----------|----------|
| OC-01 | 第 1、2、3 节：仅 `route_summary/address/eta/notes` 且固定顺序 | 文档检索 + schema 校验 |
| OC-02 | 第 4 节：二维码字段 `removed`，不属于主契约 | 字段黑名单检索 |

## 7. 对齐关系

- 本规范复用 Phase 5 的地址与降级语义边界。
- 本规范约束 `06-OUTPUT-TEMPLATE-v1.1.md` 与后续 schema/example/checklist 资产。
