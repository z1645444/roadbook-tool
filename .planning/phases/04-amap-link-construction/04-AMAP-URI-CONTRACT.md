# Phase 4 AMap URI 参数契约

## 1. 主路径与参数基线

- D-01: 主链接固定为 `https://uri.amap.com/navigation`。
- D-02: 每条导航链接必须包含 `callnative=1`。
- D-03: 当前契约默认不启用 `via`，仅保留 `from` / `to` 核心参数；未来若官方能力变化再评估。
- D-07: 分段链接必须显式填写 `from/to`，不依赖“当前位置自动补位”。

## 2. 参数编码与坐标规则

- 所有参数值执行 URL 编码，避免空格、括号、分隔符导致链接失效。
- D-04: 当坐标与文本并存时，坐标优先（GCJ-02），文本仅作为展示补充。
- 无坐标时，回退到文本参数生成可点击链接。

## 3. 多点输出形态约束

- D-05: 多途经点采用“总段 + 分段”混合输出，不把多个途经点塞进单条 URI。
- D-06: 顺序一旦确认，禁止自动重排。
- D-08: 编号固定为 `L0`（总段）与 `L1..Ln`（分段）。

## 4. 起终点回退与停止条件

- D-09: 总段起点优先级为“用户显式起点 > 当前位置”。
- D-10: 若无起点且无法定位，停止生成并要求补充起点。
- D-11: 若终点缺失，停止生成并要求补充终点。

## 5. scheme 与兼容策略

- D-12: 默认仅输出 https 链接；仅在用户明确要求时附加 scheme。
- D-13: 即使输出 scheme，仍必须保留 https 兜底。

## 6. D-01..D-13 映射表

| 决策 ID | 契约落点 | 执行说明 |
|---|---|---|
| D-01 | 主路径与参数基线 | 固定 `https://uri.amap.com/navigation` |
| D-02 | 主路径与参数基线 | 必须带 `callnative=1` |
| D-03 | 主路径与参数基线 | 默认不启用 `via` |
| D-04 | 参数编码与坐标规则 | GCJ-02 坐标优先，文本补充 |
| D-05 | 多点输出形态约束 | 总段 + 分段 |
| D-06 | 多点输出形态约束 | 禁止自动重排 |
| D-07 | 主路径与参数基线 | 分段显式 `from/to` |
| D-08 | 多点输出形态约束 | 命名固定 L0/L1..Ln |
| D-09 | 起终点回退与停止条件 | 起点优先级固定 |
| D-10 | 起终点回退与停止条件 | 缺起点且不可定位时硬停止 |
| D-11 | 起终点回退与停止条件 | 缺终点时硬停止 |
| D-12 | scheme 与兼容策略 | scheme 仅显式请求时输出 |
| D-13 | scheme 与兼容策略 | 始终保留 https 兜底 |

## 7. 需求追踪（AMAP-01..AMAP-04）

| 需求 ID | 对应条款 | 通过判定 |
|---|---|---|
| AMAP-01 | D-01, D-05, D-08 | 输出完整可点击 `https://uri.amap.com/...` |
| AMAP-02 | D-02 + 参数编码规则 | 带 `callnative=1` 且参数 URL 编码 |
| AMAP-03 | D-04 | 坐标存在时按 GCJ-02 优先 |
| AMAP-04 | D-12, D-13 | scheme 可选，https 兜底必达 |

## 8. 固定执行口径

- 输出契约以 https 为主链路，不做真实请求。
- 仅定义链接构造规则，不扩展到消息发送和二维码图片生成。
