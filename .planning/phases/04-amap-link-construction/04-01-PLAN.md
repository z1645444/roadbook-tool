<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user_intent -> link_contract | 不受信任输入进入链接参数与节点序列构造逻辑 |
| link_contract -> downstream_output | 链接构造规则被下游复用，歧义会放大成跨阶段错误 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | T | URI parameters | mitigate | 强制主路径与 `callnative=1` 固定，不允许执行阶段任意替换基线路径或移除关键参数。 |
| T-04-02 | R | order handling | mitigate | 文档中明确“确认顺序后禁止重排”，并要求 L0/L1..Ln 编号可追溯到原序列。 |
| T-04-03 | I | coordinate/text display | mitigate | 仅保留导航必要字段，坐标优先但文本只做展示补充，不引入额外敏感信息。 |
| T-04-04 | D | missing endpoint handling | mitigate | 缺失起点/终点触发停止条件与显式提示，防止生成不可用链接进入后续流程。 |
| T-04-05 | E | scheme fallback | mitigate | scheme 仅在明确请求时附带且必须保留 https fallback，避免单一路径唤起失败导致不可达。 |
</threat_model>
