# Requirements: 高德路线助手 Skill

**Defined:** 2026-04-24
**Core Value:** 在输入不完整的情况下，稳定产出可直接使用的高德地址链接；若二维码生成不可保证，则至少输出可复制地址。

## v1 Requirements

### Intent Parsing

- [x] **INT-01**: User can provide natural language demand and get parsed trip intent fields
- [x] **INT-02**: User can omit optional fields and still receive complete structured output
- [x] **INT-03**: User receives explicit assumptions and missing information in section A

### POI Recommendation

- [ ] **POI-01**: User receives 3-5 candidate POIs aligned with stated preferences
- [ ] **POI-02**: Each candidate POI includes concise recommendation reasoning
- [ ] **POI-03**: Candidate POIs are ordered to support downstream route generation

### Routing Output

- [ ] **ROUTE-01**: User receives final ordered route from start via stops to destination
- [ ] **ROUTE-02**: Each route segment includes transport mode and concise duration estimate
- [ ] **ROUTE-03**: Missing origin or destination is handled with explicit fallback assumptions

### AMap Link Contract

- [ ] **AMAP-01**: User receives a complete clickable `https://uri.amap.com/...` URL
- [ ] **AMAP-02**: URL output includes `callnative=1` and URL-encoded parameters
- [ ] **AMAP-03**: Coordinate references follow GCJ-02 assumptions when coordinates are present
- [ ] **AMAP-04**: If scheme links are provided, user also receives an `https` fallback link

### Address and QR Output

- [ ] **OUT-01**: User receives an address URL output section as the primary deliverable
- [ ] **OUT-02**: User receives QR payload text when QR generation step is available
- [ ] **OUT-03**: QR payload text is byte-identical to the address URL
- [ ] **OUT-04**: When QR cannot be guaranteed, output degrades to address-only with explanation

## v2 Requirements

### Data Quality

- **DATA-01**: User gets real-time POI scoring based on live API signals
- **DATA-02**: User gets dynamic duration adjusted by time-of-day traffic

### Artifact Generation

- **ART-01**: User gets PNG/SVG QR image artifact generated by script
- **ART-02**: User gets optional logo-embedded QR style variants
